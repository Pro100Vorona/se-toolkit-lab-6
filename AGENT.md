# Documentation Agent

## Обзор
Agent.py теперь не просто вызывает LLM, а работает как **агент** с инструментами. Он может читать файлы и просматривать директории проекта, чтобы отвечать на вопросы, используя документацию из папки `wiki`.

## Инструменты (Tools)
- **`read_file(path)`** – читает содержимое файла по относительному пути от корня проекта. Безопасность: проверяет, что путь не выходит за пределы проекта.
- **`list_files(path)`** – возвращает список файлов и папок в указанной директории (по умолчанию корень). Безопасность: аналогичная проверка.

## Агентский цикл (Agentic Loop)
1. Запрос пользователя отправляется в LLM вместе с описаниями инструментов.
2. Если LLM возвращает вызов инструментов (`tool_calls`), агент выполняет их, добавляет результаты в историю сообщений и повторяет запрос.
3. Если LLM возвращает текстовый ответ – это финальный ответ. Агент извлекает из него поле `answer` и, если есть, `source` (строка, начинающаяся с "Source:").
4. Максимальное количество вызовов инструментов за один запрос – 10.
